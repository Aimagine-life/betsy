**Цель:** Сделать Betsy user-friendly для публикации на GitHub — минимум команд, максимум понятности для обычных людей.

## 1. Концепция

Betsy — **автономный AI-агент** с настраиваемой личностью, памятью, самообучением и доступом к инструментам.

Betsy — это **агент**, а не чат-бот. Она может:
- Выполнять команды в терминале
- Читать и писать файлы
- Искать информацию в интернете через виртуальный браузер
- Устанавливать себе новые возможности
- Выполнять длительные задачи в фоне
- Подключаться к серверам по SSH
- Работать самостоятельно, пока владелец спит

**Ключевые принципы:**

- Проект для обычных людей, не для гиков
- Единственное взаимодействие с терминалом — одна команда запуска
- Всё остальное — в браузере (дашборд)
- Каналов может быть сколько угодно — все равноправны
- Betsy умеет сама добавлять новые каналы
- Весь интерфейс на русском языке

**Каналы:** Браузер (по умолчанию, не требует настройки), Telegram, Max — и любые другие. Betsy может установить новый канал сама.

**Удаляется:** весь marketplace-код, зависимости `viem`.

## 2. Agentic Loop — ядро агента

Betsy — не просто «вопрос → ответ». Она работает через многоходовой цикл с инструментами:

```
сообщение → LLM → "нужно выполнить команду" → выполняет →
→ LLM → "нужно проверить результат" → проверяет →
→ LLM → "готово, вот результат" → ответ
```

Каждый ход: LLM решает — ответить пользователю или вызвать инструмент. Цикл продолжается пока LLM не решит что задача выполнена.

### Core Tools (из коробки):

| Инструмент | Что делает |
|---|---|
| `shell` | Выполнение команд в терминале |
| `files` | Чтение / запись / список файлов |
| `http` | HTTP-запросы к API |
| `browser` | Виртуальный браузер (Playwright) — поиск, чтение сайтов, скриншоты |
| `memory` | Поиск, сохранение, удаление знаний |
| `npm_install` | Установка npm-пакетов (каналы, плагины) |
| `scheduler` | Планировщик: "напомни завтра", "проверяй сайт каждый час" |
| `self_config` | Изменение собственных настроек |
| `ssh` | Подключение к другим серверам |

### Голосовые сообщения (STT + TTS):
Betsy понимает голосовые сообщения и может отвечать голосом:
- **Входящее голосовое** → STT (OpenAI Whisper API) → текст → обработка
- **Ответ голосом** → TTS (MiniMax/OpenAI) → аудиофайл → отправка в канал
- Поддерживается в Telegram, Max, браузере
- Включается/выключается в настройках

### Безопасность инструментов:
- `shell`, `ssh` — только по разрешению владельца (можно отключить в настройках)
- **Подтверждение опасных команд** — перед выполнением деструктивных операций (`rm`, `drop`, `format`, неизвестные пакеты) Betsy спрашивает разрешение у владельца через канал:
  > «Хочу выполнить: `npm install some-package`. Разрешить? ✅/❌»
- `browser` — headless, не может взаимодействовать с GUI хоста
- `npm_install` — устанавливает только пакеты с префиксом `betsy-*` (без подтверждения), остальные — с подтверждением

### Стриминг ответов:
Пользователь видит ответ по словам, а не ждёт полный ответ:
- **Браузер:** Server-Sent Events → текст появляется в реальном времени
- **Telegram:** `sendMessageDraft` API (нативный стриминг grammY)
- **Другие каналы:** fallback — отправить полный ответ после генерации

### Прогресс длительных задач:
Если Betsy выполняет многоходовую задачу, пользователь видит статус каждого шага:
> «🔍 Открываю сайт...» → «📖 Читаю документацию...» → «✍️ Пишу код...» → «✅ Готово!»

В дашборде — страница «Задачи» с прогресс-баром для активных задач.

### Управление контекстом LLM:
Длинные разговоры не влезают в окно контекста. Стратегия:
1. Sliding window — последние N сообщений
2. Старые сообщения суммаризируются LLM в краткое описание
3. Краткое описание добавляется в системный промпт как контекст
4. Важные факты извлекаются и сохраняются в базу знаний (уже реализовано)

## 3. npm-публикация

Betsy публикуется на npm как пакет `betsy`:
```
npx betsy          # запуск без установки
npm install -g betsy && betsy   # глобальная установка
```

Установщик для VPS — отдельный пакет `betsy-install`.

## 4. Установка

### Локально
```
1. Скачай Node.js → https://nodejs.org (большая зелёная кнопка)
   Установи: Далее → Далее → Готово

2. Открой терминал:
   Windows: Win+R → cmd → Enter
   Mac: открой Terminal

3. Вставь команду:
   npx betsy

4. Откроется браузер — следуй шагам
```

### На VPS
Отдельная команда `npx betsy-install`:

```
Где установить Betsy?
○ На этом компьютере
● На сервере (VPS)

Введи IP сервера: 45.12.34.56
Введи логин: root
Введи пароль: ••••••••

🔌 Подключаюсь к серверу...
📦 Устанавливаю Node.js...
🚀 Запускаю Betsy...
✅ Готово! Открывай: http://45.12.34.56:3777
```

Установщик сам:
1. Подключается по SSH (библиотека `node-ssh`)
2. Устанавливает Node.js если нет
3. Запускает `npx betsy`
4. Настраивает автозапуск через pm2
5. Выводит URL для открытия в браузере

**Обработка ошибок:**
- Неверный пароль → понятное сообщение
- Порт 3777 закрыт → инструкция как открыть
- Нет прав sudo → инструкция что делать

## 5. Первый запуск — Визард в браузере

Терминал показывает:
```
🦀 Betsy запускается...
🌐 Открой в браузере: http://45.12.34.56:3777
```

IP определяется автоматически. Локально — браузер открывается сам.

### Шаг 1 — Привет
> «Привет! Я Betsy. Давай настроим меня за 60 секунд.»

- Поле: OpenRouter API Key
- Ссылка: «Получить бесплатно →» (openrouter.ai)
- Пояснение: «Это ключ для AI-мозга. Регистрация бесплатная.»

### Шаг 2 — Пароль
> «Придумай пароль для защиты настроек»

- Простой пароль
- Защищает дашборд от посторонних
- Шифрует API-ключи в конфиге

### Шаг 3 — Личность
> «Кто я? Давай разберёмся вместе.»

Betsy задаёт вопросы через диалог:
- «Как меня зовут?»
- «Какой у меня характер?» (пресеты + кастомный)
- «Что я умею?» (свободный ввод)

### Шаг 4 — Каналы
> «Где мне жить?»

Карточки на выбор:
- **Браузер** — включён по умолчанию, не требует настройки
- **Telegram** — нужен токен от @BotFather (пошаговая инструкция в визарде)
- **Max** — нужен токен бота (пошаговая инструкция в визарде)

Каналы можно добавить позже — прямо в чате с Betsy.

### Шаг 5 — Готово!
> «Я готова! Поговори со мной.»

- Чат прямо в браузере
- Ссылки на подключённые каналы

## 6. Архитектура

```
betsy/
├── core/           ← Ядро: LLM, agentic loop, память, личность, самообучение
│   ├── engine.ts   ← Agentic loop (multi-turn tool use)
│   └── tools/      ← Core tools (shell, files, browser, memory, ...)
├── channels/       ← Адаптеры каналов (все равноправны)
│   ├── telegram/
│   ├── max/
│   └── browser/    ← Чат в дашборде (WebSocket)
├── plugins/        ← Расширения: голос, видео, селфи
└── ui/             ← Веб-визард + дашборд + чат (React)
```

### Channel Interface
```typescript
interface Channel {
  name: string
  requiredConfig: string[]
  start(config: Record<string, string>): Promise<void>
  stop(): Promise<void>
  send(userId: string, message: OutgoingMessage): Promise<void>
  onMessage(handler: (msg: IncomingMessage) => Promise<OutgoingMessage>): void
}
```

Core ничего не знает о каналах. Каждый канал — отдельный адаптер с единым интерфейсом. Новый канал = новый файл в `channels/`.

Browser-канал работает через WebSocket (`ws://IP:3777/chat`).

### Tool Interface
```typescript
interface Tool {
  name: string
  description: string
  parameters: Record<string, ToolParam>
  execute(params: Record<string, unknown>): Promise<ToolResult>
}
```

## 7. Конфигурация

Хранится в `~/.betsy/config.yaml` (домашняя папка). Генерируется визардом. Пользователь никогда не редактирует YAML вручную.

```yaml
# ~/.betsy/config.yaml (генерируется автоматически)
agent:
  name: Бетси
  personality:
    tone: friendly
    style: detailed
    custom_instructions: "..."

security:
  password_hash: "pbkdf2:sha256:..."
  tools:
    shell: true        # разрешить выполнение команд
    ssh: false         # SSH отключен по умолчанию
    browser: true      # виртуальный браузер
    npm_install: true  # установка пакетов

llm:
  provider: openrouter
  api_key: "encrypted:AES256:..."
  fast_model: google/gemini-2.5-flash
  strong_model: anthropic/claude-sonnet-4

channels:
  browser:
    enabled: true
  telegram:
    enabled: true
    token: "encrypted:AES256:..."
    owner_chat_id: 123456789
  max:
    enabled: false

plugins:
  voice:
    enabled: false
  video:
    enabled: false
  selfies:
    enabled: false
```

API-ключи шифруются AES-256 через мастер-пароль. В интерфейсе показываются замаскированными: `sk-or-v1-••••••••...abc`.

## 10. Самодобавление каналов

Пользователь пишет Betsy:
> «Подключи меня к Discord»

Betsy (используя agentic loop + tools):
1. Ищет `betsy-channel-discord` в npm (tool: `shell` → `npm search`)
2. **Найден** → устанавливает (tool: `npm_install`), запрашивает настройки, подключает
3. **Не найден:**
   > «Не нашла готового плагина. Пришли мне документацию API — попробую написать сама.»
4. Пользователь присылает ссылку → Betsy открывает в виртуальном браузере (tool: `browser`), читает доку
5. Генерирует код канала (tool: `files` → пишет файл), тестирует (tool: `shell` → запускает тест)
6. **Тест прошёл** → устанавливает
7. **Не получилось:**
   > «Не смогла разобраться. Вот ссылка — попроси разработчиков: https://github.com/Aimagine-life/betsy/issues/new»

## 11. Скиллы (навыки)

Betsy умеет создавать и использовать скиллы — готовые сценарии для повторяющихся задач.

### Пользователь просит:
> «Научись проверять мой сайт на ошибки»

Betsy:
1. Уточняет задачу: «Какой сайт? Что проверять? Как часто?»
2. Составляет скилл — пошаговый сценарий с использованием tools:
   ```yaml
   # ~/.betsy/skills/check-website.yaml
   name: Проверка сайта
   trigger: "проверь сайт" | scheduler: "каждые 6 часов"
   steps:
     - tool: browser
       action: open https://mysite.com
     - tool: browser
       action: screenshot
     - tool: http
       action: check status codes for all links
     - tool: memory
       action: compare with previous check
     - notify: owner
       message: "Результат проверки: {summary}"
   ```
3. Тестирует скилл
4. Сохраняет

### Управление:
- «Покажи мои скиллы» → список с описаниями
- «Удали скилл проверки сайта»
- «Запусти проверку сайта» → ручной запуск
- Скиллы можно привязать к планировщику (tool: `scheduler`)

### Встроенные скиллы (из коробки):
- **Мониторинг сайта** — проверка доступности + изменения контента
- **Ежедневная сводка** — что произошло за день, статистика
- **Создание скилла** — мета-скилл: Betsy умеет составлять новые скиллы из описания на русском языке

## 12. Самообучение

Betsy учится в фоне. Пользователь ничего не делает.

- Каждые 30 минут — анализ разговоров
- Выделяет факты, предпочтения, паттерны
- Сохраняет в SQLite с FTS5 (BM25-поиск)
- Использует при следующих разговорах

**В дашборде:**
```

📚 Последняя учёба: 12 минут назад
```

**В чате:**
- «Что ты про меня знаешь?»
- «Забудь про мои предпочтения по еде»

## 12. Деплой

Polling работает везде — локально, на VPS, без домена.

**Автозапуск на VPS** (настраивается установщиком через pm2):
```
pm2 start betsy --name betsy
pm2 startup
pm2 save
```

### Docker (альтернативный способ):
```bash
docker run -d --name betsy \
  -p 3777:3777 \
  -v betsy-data:/root/.betsy \
  aimagine/betsy
```
Dockerfile включён в репо. Образ публикуется на Docker Hub.

### Playwright (виртуальный браузер):
При первом запуске Betsy скачивает браузер Chromium (~150MB). Прогресс показывается:
```
📥 Скачиваю виртуальный браузер... 45% [███████░░░░░░░]
```
На VPS с мало оперативной памяти (<1GB) — предупреждение о возможных проблемах.

## 14. Диагностика

Всё в дашборде. Никаких CLI-команд.

**Страница «Статус»:**
```
✅ LLM: работает (OpenRouter)
✅ Telegram: подключён
❌ Max: ошибка — неверный токен → [Исправить]
🧠 Память: 47 записей, 2.3MB
🔧 Инструменты: shell ✅, browser ✅, ssh ❌
```

При ошибках — уведомление во все подключённые каналы.

### Отслеживание расходов:
```
💰 Сегодня: $0.12 | Этот месяц: $3.45 | Всего: $15.20
📊 По моделям: Gemini Flash $2.10, Claude Sonnet $1.35
```
Betsy считает токены каждого запроса, умножает на стоимость модели (берёт из OpenRouter API). Показывает в дашборде на главной.
